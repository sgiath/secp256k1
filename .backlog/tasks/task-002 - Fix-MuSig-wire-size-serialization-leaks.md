## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
> *This was generated by AI during triage.*

Finding: MuSig NIFs allocate and return struct-sized binaries while libsecp256k1 serializers write smaller fixed wire formats. `nonce_gen/5` returns a 132-byte pubnonce even though the serializer writes 66 bytes. `nonce_agg/1` returns a 132-byte aggnonce even though the serializer writes 66 bytes. `partial_sign/4` returns a 36-byte partial signature even though the serializer writes 32 bytes. The unwritten tail can contain uninitialized BEAM/NIF memory.

Evidence checked:
- `c_src/musig.c` allocates `sizeof(pubnonce)`, `sizeof(aggnonce)`, and `sizeof(partial_sig)` before calling serializers.
- `c_src/secp256k1/include/secp256k1_musig.h` documents 66-byte pubnonce/aggnonce serialization and 32-byte partial signature serialization.
- Focused probe observed nonzero tail bytes including BEAM path-like data after serialized MuSig outputs.
- `lib/secp256k1/musig.ex` type docs currently mix serialized and opaque struct sizes.

Expected direction:
- Allocate exact serialized wire sizes: 66 bytes for pubnonce, 66 bytes for aggnonce, 32 bytes for partial signatures.
- Enforce exact input lengths before parse calls.
- Update Elixir types/docs and tests to match wire sizes.
- Add regression tests proving returned binaries have no extra tail bytes.

## Implementation Brief

Root cause: `c_src/musig.c` allocates BEAM binaries using libsecp256k1 opaque C struct sizes, then calls serializers that write smaller canonical wire formats. The unwritten tail is returned to Elixir and can contain uninitialized NIF/BEAM memory.

Use the upstream header as source of truth, but do not edit it. The relevant serializer contracts are 66 bytes for `secp256k1_musig_pubnonce_serialize`, 66 bytes for `secp256k1_musig_aggnonce_serialize`, and 32 bytes for `secp256k1_musig_partial_sig_serialize`.

Concrete edits:
- Add local wire-size constants near the top of `c_src/musig.c`: 66-byte pubnonce, 66-byte aggnonce, 32-byte partial signature.
- In `nonce_gen/5`, allocate the public nonce binary with the 66-byte wire-size constant before `secp256k1_musig_pubnonce_serialize`.
- In `nonce_agg/1`, require every input pubnonce to be exactly 66 bytes before `secp256k1_musig_pubnonce_parse`, then allocate the aggregate nonce result with the 66-byte wire-size constant.
- In `nonce_process/3`, require the aggregate nonce input to be exactly 66 bytes before `secp256k1_musig_aggnonce_parse`.
- In `partial_sign/4`, allocate the partial signature result with the 32-byte wire-size constant before `secp256k1_musig_partial_sig_serialize`.
- In `partial_sig_verify/5`, require the partial signature input to be exactly 32 bytes and the public nonce input to be exactly 66 bytes before parse calls.
- In `partial_sig_agg/2`, require every partial signature input to be exactly 32 bytes before `secp256k1_musig_partial_sig_parse`.
- Update `lib/secp256k1/musig.ex` types/comments so `aggnonce` is 66 bytes and `partial_sig` is 32 bytes. `pubnonce` already has the correct type size.
- Keep `keyagg_cache` and `session` as opaque struct-sized binaries for this task. Malformed opaque-state abort hardening belongs to `TASK-003`.
- Do not touch secnonce race behavior here. Concurrency-safe nonce consumption belongs to `TASK-004`.

## Testing Requirements

Add or update tests so the current bug fails before the C fix:
- In the main MuSig signing-flow test, assert all generated public nonces are 66 bytes, the aggregate nonce is 66 bytes, and each partial signature is 32 bytes.
- In MuSig vector tests, assert `MuSig.nonce_agg/1 == expected` directly. Remove the existing `byte_size(aggnonce) == 132` and `binary_part(aggnonce, 0, 66)` workaround.
- Add regression tests for overlong serialized inputs:
  - `MuSig.nonce_agg([pubnonce <> <<0>>])` raises `ArgumentError`.
  - `MuSig.nonce_process(aggnonce <> <<0>>, msg, cache)` raises `ArgumentError`.

  - `MuSig.partial_sig_agg(session, [partial_sig <> <<0>>])` raises `ArgumentError`.
- Include short-input coverage for at least one pubnonce, aggnonce, and partial signature path to prove exact-length checking, not just overlong rejection.
