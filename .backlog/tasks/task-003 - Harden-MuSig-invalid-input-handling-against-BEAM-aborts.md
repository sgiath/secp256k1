## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
> *This was generated by AI during triage.*

Finding: malformed MuSig opaque inputs can abort the BEAM process through libsecp256k1 illegal-argument callbacks. The wrapper accepts raw cache/session-sized binaries, copies them into opaque structs, and passes them to libsecp256k1. Invalid contents can hit `ARG_CHECK` paths whose default callback aborts. `nonce_gen/5` also documents `pubkey` as optional, but the wrapped C API requires it and aborts when `nil` is passed.

Evidence checked:
- `c_src/musig.c` accepts raw `keyagg_cache` and `session` binaries by size only before calling libsecp256k1 functions.
- `lib/secp256k1/musig.ex` documents `pubkey` as optional for `nonce_gen/5`.
- `secp256k1_musig_nonce_gen` requires a non-null pubkey in the upstream header.
- Focused probes showed `MuSig.pubkey_get(<<0::197*8>>)` and `MuSig.nonce_process(<<0::66*8>>, msg, <<0::197*8>>)` aborting the process via libsecp256k1 illegal argument output.
- `MuSig.nonce_gen(seckey, nil, msg, cache, nil)` also aborted with `pubkey != NULL`.

Expected direction:
- Install non-aborting libsecp256k1 illegal/error callbacks for NIF contexts, with enough context logged or surfaced to debug failures.
- Reject impossible API combinations in Elixir or C before entering libsecp256k1.
- Prefer resources or validated wrappers for opaque cache/session state instead of accepting arbitrary binaries where possible.
- Add subprocess-based regression tests for abort-prone malformed inputs.

## Implementation Plan

## Implementation Brief

Root-cause break: stop exposing libsecp256k1 opaque MuSig structs as plain binaries. Upstream guarantees sizes for `secp256k1_musig_keyagg_cache` and `secp256k1_musig_session`, but says they have no serialization/parsing functions. Treat them like process-local NIF state.

Do not try to preserve compatibility by checking internal magic bytes on caller-provided binaries. That would keep unsupported opaque structs as a public wire format and rely on private upstream layout details. The supported boundary is: serialized MuSig values with upstream parse/serialize APIs remain binaries; opaque cache/session state becomes NIF resources.

Current abort-prone paths:
- `pubkey_get/1`, `pubkey_ec_tweak_add/2`, `pubkey_xonly_tweak_add/2`, `nonce_gen/5`, `nonce_process/3`, `partial_sign/4`, and `partial_sig_verify/5` copy raw cache binaries into `secp256k1_musig_keyagg_cache`.
- `partial_sign/4`, `partial_sig_verify/5`, and `partial_sig_agg/2` copy raw session binaries into `secp256k1_musig_session`.
- `nonce_gen/5` allows `pubkey == NULL`, but upstream marks the `pubkey` argument non-null.

## Concrete Edits

1. Add resource wrappers in `c_src/musig.c`.
   - Add `keyagg_cache_resource_type` storing `secp256k1_musig_keyagg_cache`.
   - Add `session_resource_type` storing `secp256k1_musig_session`.
   - Keep existing `secnonce_resource_type`.
   - Open all three resource types in `musig_load/3` and `musig_upgrade/4`; fail load/upgrade if any resource type cannot be opened.
   - If resource setup fails after creating the libsecp256k1 context, destroy the context before returning failure.
   - Destructors should zero resource memory. `keyagg_cache` and `session` are not secret, but zeroing avoids stale opaque state and matches current NIF hygiene.

2. Return resources instead of opaque binaries.
   - `pubkey_agg/1`: allocate a keyagg cache resource, copy the valid `cache` into it, return `{:ok, agg_xonly_pubkey, cache_resource}`.
   - `pubkey_ec_tweak_add/2`: require input cache resource, copy it into a new cache resource, apply tweak to the new cache, return `{:ok, new_cache_resource, tweaked_pubkey}`. Do not mutate the input resource.
   - `pubkey_xonly_tweak_add/2`: same functional-copy behavior as EC tweak.
   - `nonce_process/3`: require cache resource, allocate a session resource, write the produced session into it, return the resource.

3. Consume resources instead of raw opaque binaries.
   - `pubkey_get/1`: require keyagg cache resource.
   - `nonce_gen/5`: accept `nil` or keyagg cache resource for cache, because upstream allows cache to be optional. Reject any non-`nil`, non-resource cache.
   - `partial_sign/4`: require secnonce resource, keyagg cache resource, and session resource.
   - `partial_sig_verify/5`: require keyagg cache resource and session resource.
   - `partial_sig_agg/2`: require session resource.
   - Any old raw `197`-byte cache or `133`-byte session binary should hit `badarg` before libsecp256k1.

4. Fix `nonce_gen/5` public contract.
   - Require `pubkey` to be a valid compressed or uncompressed public key binary in C before calling `secp256k1_musig_nonce_gen`.
   - Reject `nil` `pubkey` with `badarg` / `ArgumentError`, matching the existing invalid-argument style.
   - Keep `seckey`, `msg`, `cache`, and `extra` optional where upstream permits them, but accept only `nil` or the exact expected type/size. Reject non-`nil` bad terms instead of silently treating failed `enif_inspect_binary` / `enif_get_resource` checks as omitted arguments.
   - Update `lib/secp256k1/musig.ex` docs/specs and NIF stubs: `pubkey` is required; cache/session/secnonce are resources and not serializable.

5. Install non-aborting libsecp256k1 callbacks in `c_src/utils.h`.
   - After `secp256k1_context_create`, call `secp256k1_context_set_illegal_callback(ctx, callback, NULL)` and `secp256k1_context_set_error_callback(ctx, callback, NULL)`.
   - Callback must not abort. It should log the callback kind and libsecp256k1 message to stderr with a stable prefix such as `[libsecp256k1 NIF] illegal argument: ...`.
   - Install callbacks in both `load/3` and `upgrade/4`.
   - While touching `load/3`, replace the current `assert(return_val)` path with checked cleanup/return so context randomization failure cannot abort the VM.
   - Keep function-level validation/resource conversion as the primary fix; callbacks are the safety net for missed checks.

6. Preserve wire-format behavior from `TASK-002`.
   - Public nonces stay 66-byte serialized binaries.
   - Aggregate nonces stay 66-byte serialized binaries.
   - Partial signatures stay 32-byte serialized binaries.
   - Only cache/session move to resources.

## Reproduction / Verification Commands

Run each command in a separate shell process. Before the fix, these commands can abort the child BEAM VM and print `[libsecp256k1] illegal argument: ...`. After the fix, each command should complete without VM abort/signal, either by raising `ArgumentError` or by returning a clear `{:error, reason}` result.

Malformed key aggregation cache:

```bash
mix run -e 'alias Secp256k1.MuSig; IO.inspect(MuSig.pubkey_get(<<0::197*8>>))'
```

Malformed cache through nonce processing:

```bash
mix run -e 'alias Secp256k1.MuSig; msg = <<1::32*8>>; IO.inspect(MuSig.nonce_process(<<0::66*8>>, msg, <<0::197*8>>))'
```

Missing required signer public key:

```bash
mix run -e 'alias Secp256k1.MuSig; {sec, pub} = Secp256k1.keypair(:compressed); {:ok, _, cache} = MuSig.pubkey_agg([pub]); msg = <<1::32*8>>; IO.inspect(MuSig.nonce_gen(sec, nil, msg, cache, nil))'
```

Malformed cache through tweak paths:

```bash
mix run -e 'alias Secp256k1.MuSig; IO.inspect(MuSig.pubkey_ec_tweak_add(<<0::197*8>>, <<1::32*8>>))'
mix run -e 'alias Secp256k1.MuSig; IO.inspect(MuSig.pubkey_xonly_tweak_add(<<0::197*8>>, <<1::32*8>>))'
```

Malformed session through signature aggregation:

```bash
mix run -e 'alias Secp256k1.MuSig; {sec, pub} = Secp256k1.keypair(:compressed); {:ok, _, cache} = MuSig.pubkey_agg([pub]); msg = <<1::32*8>>; {:ok, secnonce, pubnonce} = MuSig.nonce_gen(sec, pub, msg, cache, nil); aggnonce = MuSig.nonce_agg([pubnonce]); session = MuSig.nonce_process(aggnonce, msg, cache); partial_sig = MuSig.partial_sign(secnonce, sec, cache, session); IO.inspect(MuSig.partial_sig_agg(<<0::133*8>>, [partial_sig]))'
```

Normal-flow guardrail:


Add focused in-process tests:

- Tweak flow still works and returns a new cache resource.
- `pubkey_get(cache_resource)` returns a compressed public key.
- Old forged cache/session binaries are rejected:
  - `MuSig.pubkey_get(<<0::197*8>>)`

## Out Of Scope

- Do not add cache/session serialization APIs. Upstream has no parser/serializer contract for those structs.
- Do not change public nonce, aggregate nonce, or partial signature wire sizes.
- Do not solve secnonce concurrent consumption races here; that belongs to `TASK-004`.
- Do not add a counter-based nonce-generation API unless needed for this fix.
