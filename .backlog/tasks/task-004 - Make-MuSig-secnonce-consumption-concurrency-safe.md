## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
> *This was generated by AI during triage.*

Finding: MuSig nonce reuse protection is raceable. `secnonce_wrapper.used` is a plain `int`, checked before signing and set only after `secp256k1_musig_partial_sign` succeeds. Concurrent callers using the same secnonce resource can both observe `used == 0` and enter the signing call. Because MuSig nonce reuse leaks the signing key, the consumption check must be atomic.

Evidence checked:
- `c_src/musig.c` defines `secnonce_wrapper` with `int used`.
- `partial_sign` checks `wrapper->used` before creating the keypair and signing.
- `wrapper->used = 1` is assigned only after the libsecp256k1 partial signing call succeeds.
- The resource destructor securely erases the wrapper, but it does not serialize concurrent access while the resource is live.

Expected direction:
- Protect secnonce consumption with a mutex or atomic compare-and-set.
- Mark the nonce consumed before entering signing, or otherwise ensure exactly one caller can ever use it.
- Preserve secure erase behavior and clear keypair/secnonce state on all exit paths.
- Add a concurrent regression test that races repeated `partial_sign/4` calls and proves only one can succeed.

## Implementation Plan

1. Add per-resource nonce consumption lock in `c_src/musig.c`.
   - Extend `secnonce_wrapper` directly with `ErlNifMutex *mutex` plus `used`.
   - No backward compatibility required: this is not deployed anywhere, so do not add hot-upgrade bridges, old resource layout handling, migration code, or resource type versioning just for compatibility.

2. Initialize and destroy lock with the secnonce resource.
   - In `nonce_gen/5`, create the mutex after resource allocation and before returning the resource term.
   - On setup failure, release binary/resource state and secure-erase generated secnonce data.
   - In `destruct_secnonce`, destroy the mutex first, then secure-erase nonce/wrapper memory.

3. Consume secnonce before signing.
   - In `partial_sign/4`, validate secnonce/cache/session resources and create the keypair first.
   - Lock the secnonce mutex, check `used`, set `used = 1`, unlock.
   - If already used, erase the keypair and return `{:error, "nonce already used"}`.
   - Do not roll `used` back after it flips; upstream may have read/zeroed the nonce even when signing fails.

4. Keep sensitive cleanup explicit.
   - Erase `keypair` on every exit after keypair creation.
   - After `secp256k1_musig_partial_sign` returns, explicitly secure-erase `wrapper->nonce` even though upstream also zeroes it.
   - Preserve existing error shapes where practical.

5. Add concurrent regression coverage in `test/secp256k1/musig_test.exs`.
   - Generate one valid MuSig signing state with one shared `secnonce`.
   - Barrier-release many `Task.async` workers calling `MuSig.partial_sign/4` with the same resource.
   - Assert exactly one result is a 32-byte partial signature.
   - Assert every loser returns `{:error, "nonce already used"}`; no loser should enter upstream and return `"secp256k1_musig_partial_sign failed"`.

6. Validate.
   - Run `mix test test/secp256k1/musig_test.exs`.
   - Run `mix check` if the focused test passes.

## Final Summary

<!-- SECTION:FINAL_SUMMARY:BEGIN -->
Implemented concurrency-safe MuSig secnonce consumption.

Changed `c_src/musig.c` so each secnonce resource owns an `ErlNifMutex`. `partial_sign/4` now validates resources and the keypair, then atomically checks and flips `used` under the mutex before calling `secp256k1_musig_partial_sign`. The nonce is never rolled back after consumption, and both keypair and secnonce state are explicitly erased on signing success and signing failure paths.

Added a concurrent regression test in `test/secp256k1/musig_test.exs` that releases 32 tasks against one shared secnonce and asserts exactly one 32-byte partial signature succeeds while all other callers receive `{:error, "nonce already used"}`.

Updated `CHANGELOG.md` with the unreleased MuSig2 concurrency-safety fix.
