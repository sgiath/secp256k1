## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
> *This was generated by AI during triage.*

Finding: MuSig nonce reuse protection is raceable. `secnonce_wrapper.used` is a plain `int`, checked before signing and set only after `secp256k1_musig_partial_sign` succeeds. Concurrent callers using the same secnonce resource can both observe `used == 0` and enter the signing call. Because MuSig nonce reuse leaks the signing key, the consumption check must be atomic.

Evidence checked:
- `c_src/musig.c` defines `secnonce_wrapper` with `int used`.
- `partial_sign` checks `wrapper->used` before creating the keypair and signing.
- `wrapper->used = 1` is assigned only after the libsecp256k1 partial signing call succeeds.
- The resource destructor securely erases the wrapper, but it does not serialize concurrent access while the resource is live.

Expected direction:
- Protect secnonce consumption with a mutex or atomic compare-and-set.
- Mark the nonce consumed before entering signing, or otherwise ensure exactly one caller can ever use it.
- Preserve secure erase behavior and clear keypair/secnonce state on all exit paths.
- Add a concurrent regression test that races repeated `partial_sign/4` calls and proves only one can succeed.
